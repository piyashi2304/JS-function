Q5. "Print output: var x = 21; girl (); console.log(x) function girl() { console.log(x); var x = 20; }; "

Reason:Inside the girl function: console.log(x) will print undefined because the variable x is hoisted to the top of the function scope. However, at this point, it has not yet been assigned a value, so it's undefined. var x = 20, A new local variable x is declared within the function scope and assigned the value 20. This local variable shadows the outer x variable within the function scope. console.log(x) will print 21 because it's referencing the outer x variable that was declared at the beginning of the code, outside of the girl function. The value of this outer x was not changed within the function.
